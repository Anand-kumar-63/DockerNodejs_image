# Node.js + MongoDB REST API

A simple Express.js REST API written in TypeScript that connects to MongoDB. Supports running **locally** and inside a **Docker container**.

---

## Tech Stack

- **Runtime:** Node.js v22
- **Language:** TypeScript (compiled to ESM)
- **Framework:** Express.js v5
- **Database:** MongoDB (via Mongoose)
- **Containerization:** Docker

---

connect_to_mongodb/

## Environment Variables

Create a `.env` file in the project root (see `.env` for reference):

```env
PORT=4000
MONGO_URL=mongodb://localhost:27017/myDatabase
```

| Variable    | Description                        | Default                                   |
|-------------|------------------------------------|-------------------------------------------|
| `PORT`      | Port the server listens on         | `4000`                                    |
| `MONGO_URL` | Full MongoDB connection string     | `mongodb://localhost:27017/myDatabase`    |

## Running Locally

### Prerequisites

- [Node.js v18+](https://nodejs.org/)
- [MongoDB](https://www.mongodb.com/try/download/community) running locally **or** a [MongoDB Atlas](https://www.mongodb.com/atlas) URI

### Steps

**1. Clone the repository**

```bash
git clone <repository-url>
cd connect_to_mongodb
```

**2. Install dependencies**

```bash
npm install
```

**3. Set up environment variables**

```bash
# Create your .env file
cp .env .env.local
```

Edit `.env` and set your `MONGO_URL`:

```env
PORT=4000
MONGO_URL=mongodb://localhost:27017/myDatabase
```

**4. Start in development mode** (auto-reloads on changes)

```bash
npm run dev
```

**5. Or build and start in production mode**

```bash
npm run build
npm start
```

The server will be running at: `http://localhost:4000`

## Running with Docker

### Prerequisites

- [Docker Desktop](https://www.docker.com/products/docker-desktop/) installed and running
- MongoDB accessible (locally or via Atlas)

---

### Option A — Pull from Docker Hub & Run

The image is published on Docker Hub as `nexus916/nodejsimage_dockerhub_3`.

**Connect to MongoDB on your host machine:**

```bash
docker run -p 4000:4000 \
  -e MONGO_URL="mongodb://host.docker.internal:27017/myDatabase" \
  nexus916/nodejsimage_dockerhub_3:v8
```

> `host.docker.internal` is Docker's special hostname that resolves to your **host machine's IP** from inside a container. Use this when MongoDB is running locally on your PC.

**Connect to MongoDB Atlas:**

```bash
docker run -p 4000:4000 \
  -e MONGO_URL="mongodb+srv://<user>:<password>@<cluster>.mongodb.net/myDatabase" \
  nexus916/nodejsimage_dockerhub_3:v8
```

---

### Option B — Build the Image Locally & Run

**1. Build the image**

```bash
docker build -t my-node-app .
```

**2. Run the container**

```bash
docker run -p 4000:4000 \
  -e MONGO_URL="mongodb://host.docker.internal:27017/myDatabase" \
  my-node-app
```

---

### Option C — Docker Compose (App + MongoDB together)

Create a `docker-compose.yml` in the project root:

```yaml
version: "3.8"

services:
  app:
    image: nexus916/nodejsimage_dockerhub_3:v8
    ports:
      - "4000:4000"
    environment:
      - MONGO_URL=mongodb://mongo:27017/myDatabase
      - PORT=4000
    depends_on:
      - mongo

  mongo:
    image: mongo:7
    ports:
      - "27017:27017"
    volumes:
      - mongo_data:/data/db

volumes:
  mongo_data:
```

Then run:

```bash
docker compose up
```

## Verifying It Works

Once the server is running (locally or in Docker), test it with:

```bash
# Health check
curl http://localhost:4000/

# Create a user
curl -X POST http://localhost:4000/Createuser \
  -H "Content-Type: application/json" \
  -d '{"name": "Alice", "email": "alice@example.com", "age": 30}'
```

## Common Issues

| Problem | Cause | Fix |
|---------|-------|-----|
| `MongoDb connection error` in Docker | Using `localhost` inside container | Use `host.docker.internal` as the MongoDB host |
| `ERR_MODULE_NOT_FOUND` | Missing `.js` extensions in ESM imports | Ensure `tsconfig.json` uses `"module": "NodeNext"` |
| `Cannot find image` | Image not built yet | Run `docker build` first |
| `Port already in use` | Another process on port 4000 | Change `-p 4001:4000` to use a different host port |
